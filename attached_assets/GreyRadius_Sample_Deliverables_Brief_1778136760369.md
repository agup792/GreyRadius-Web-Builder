# GreyRadius — Sample Deliverables Brief for Replit

**Companion document to the main Replit prompt and asset brief.** This document gives a detailed review of the 22 sample deliverables in the existing Tech deck (Pages 9, 11, 15, 17, 18, 19), maps each one to the new website's 6 offerings, and provides ready-to-use captions and deployment instructions.

The objective: when a buyer lands on the website, they should see *exactly* what GreyRadius outputs look like. Not just claims — actual artefacts.

---

## 1. WHY THIS MATTERS

Buyers comparing GreyRadius against Big-4 or boutique competitors scan service pages in 90 seconds. The single fastest credibility signal is showing the *form factor* of actual deliverables — frameworks, matrices, scoring grids, financial models, org charts. This deck has 22 of them. Every one is anonymised and ready to display publicly.

Three rules for using these on the website:

1. **Always caption with the buyer's question.** Not "Opportunity Heat Map" — but "How we score opportunities by impact and feasibility." The visitor is asking "what would this look like for me?"
2. **Show 4-6 per service page in a clickable gallery, not all 22.** Visitors don't scroll through 22 thumbnails. Select the most relevant ones per page.
3. **Watermark each thumbnail with "Sample · GreyRadius" in a corner.** Confirms anonymisation, prevents misuse if screenshotted.

### PAGE 9 — Opportunity Mapping & Prioritization (2 snapshots)

#### S9.1 — Opportunity Assessment Matrix (Heat Map)

**What it shows:** A scoring matrix with 10+ rows of products/initiatives scored across multiple weighted dimensions (Time saving impact, Cost savings, Customer satisfaction, Feasibility, Strategic fit, Sustainability rating). Cells colour-coded green / yellow / red. Each row gets a final weighted score and a strategic ranking.

**The framework:** Weighted multi-criteria scoring model. GreyRadius defines the parameters with the client during scoping, weights each dimension, then scores every candidate opportunity using primary research data + analyst judgment.

**The buyer's question this answers:** *"How do I rank 10-15 potential opportunities against each other when each one has different upside and different feasibility?"*

#### S9.2 — Product Opportunity Mapping Matrix (Bubble Chart)

**What it shows:** A 2×2 quadrant chart with Impact on the Y-axis (Low → High) and Feasibility on the X-axis (Low → High). Bubbles sized by addressable customer base. Each bubble is a discrete product/feature opportunity, labelled with implementation timeline (e.g., "5 months", "12 months", "Setting up new interior designing feature", "Customer grievances and support for multiple housing projects", "Maintenance & Addition Project Management (MAPM)", "Online loan applications tied with banks", "Construction development status updates", "Launch online application processing and Onboarding", "Adding online discounts portal for new housing").

The four quadrants:
- **Quick Wins** (top-right) — High Impact, High Feasibility — easy to implement, drives significant benefits quickly
- **Major Projects** (top-left) — High Impact, Low/Medium Feasibility — high-potential features needing investment, regulatory approvals, or infrastructure
- **Fill-ins** (bottom-right) — Medium Impact, High Feasibility — easy to implement within less timeline, low-to-medium impact
- **Moonshots / Beyond direct control** (bottom-left) — Low Impact, Low Feasibility — complex products

**The framework:** Impact-feasibility quadrant analysis with bubble size as third dimension (customer base reach). This is a classic 2x2 done with primary-research-validated data.

**The buyer's question this answers:** *"Which 3-5 opportunities should I pursue first, second, and never?"*
