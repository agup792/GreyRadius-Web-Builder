## SECTION 1 — CRITICAL ROUTING AND STRUCTURAL BUGS

These are not cosmetic issues. They break navigation and SEO simultaneously.

### 1.1 Fix the /about/ directory routing [CRITICAL — PRE-LAUNCH]

**Problem:** Navigating to `/about/` currently renders the full homepage instead of an About page. Every nav link from within the `/about/` directory is also broken because relative paths prepend `/about/` to all hrefs (e.g. Services links to `/about/services/`, Industries links to `/about/industries/` — all 404).

**Fix required:**
- Create `/about/index.html` as a proper standalone About page (not a copy of the homepage).
- Switch all nav template hrefs from document-relative to root-relative paths. Every `href="services/..."` inside the `/about/` directory should be `href="/services/..."`. This applies to every template partial that generates navigation.
- Verify: after fix, clicking "About" from any page should land on a distinct About page, and all nav links from that page should route correctly.

### 1.2 Fix the /case-studies/ directory routing [CRITICAL — PRE-LAUNCH]

**Problem:** Same as 1.1. The `/case-studies/` URL serves the homepage instead of a case studies listing page. All nav links from within that directory are broken with `/case-studies/` prepended.

**Fix required:** Same root-relative path fix as 1.1. Build a proper `/case-studies/index.html` listing page.

### 1.3 Fix all canonical tags [CRITICAL — PRE-LAUNCH]

**Problem:** Every page currently has `canonical: https://greyradius.com/` pointing to the old site. When the new site launches on its production domain, Google will read these canonicals and continue crediting all content to the old domain. All new content rankings will be lost.

**Fix required:**
- On launch, update every page's canonical tag to its own correct URL on the new domain.
- Format: `<link rel="canonical" href="https://greyradius.com/services/opportunity-assessment.html" />` (using the final production domain, not the Replit dev URL).
- This must be templated so it auto-generates per-page — not hardcoded per page manually.

### 1.4 Fix the /industries/technology.html 404 [CRITICAL — PRE-LAUNCH]

**Problem:** The URL `/industries/technology.html` returns a 404. This URL is linked from the main nav and from multiple internal pages. Visitors clicking "Technology" under Industries hit a dead page.

**Fix required:** Build all 6 industry pages (see Section 7 for content spec). The page files must exist before launch:
